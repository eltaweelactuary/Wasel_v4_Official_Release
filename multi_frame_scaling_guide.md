# دليل زيادة الموارد لتشغيل الترجمة الحركية المتقدمة (Motion Video Capture)

النسخة الحالية من تطبيق "مصر الرقمية" (العميل) تعمل بـ **إطار واحد (Single Frame)** لضمان سرعة رد فعل فائقة (Ultra-fast response) تتناسب مع موارد الخادم المجانية أو المحدودة الحالية على Google Cloud Run.

ولكن، لغة الإشارة تعتمد بشكل أساسي على **الحركة (Motion)**. لتحسين دقة الترجمة وجعلها ترصد حركات الأيدي المعقدة، يجب العودة للعمل بتسلسل إطارات (Multi-frame Array).

إليك خطوات تطبيق ذلك بالتفصيل بمجرد زيادة موارد الخادم.

---

## 🚀 1. التعديل البرمجي المطلوب في واجهة المستخدم (العميل)

للالتقاط المستمر كفيديو مصغر وإرساله للمحرك، يجب فتح الملف `Digital_Egypt_Sign_Client/templates/index.html` وتعديل دالة الـ `loop()` فيه واستبدالها بهذا الكود السحري:

```javascript
// === الكود المتقدم لالتقاط 3 إطارات للحركة ===
function loop() {
    if (!running) return;
    if (botBusy) { setTimeout(loop, 500); return; }

    var frames = [];
    var captureIndex = 0;

    function captureNext() {
        if (!running) return;
        
        // رسم الإطار من الكاميرا
        cx.drawImage(vid, 0, 0, 512, 512);
        
        // تحويله لـ Base64 مضغوط (جودة 50% لتقليل الحجم)
        frames.push(cv.toDataURL('image/jpeg', 0.5));
        captureIndex++;

        if (captureIndex < 3) {
            // انتظار 300 جزء من الثانية قبل الإطار التالي لاصطياد الحركة
            setTimeout(captureNext, 300);
        } else {
            // تم التقاط 3 إطارات تغطي ثانية كاملة = فيديو مصغر
            // إرسال المصفوفة للمحرك
            sendFrames(frames);
        }
    }
    captureNext();
}
```

## ☁️ 2. المتطلبات السحابية (GCP Resources)

السبب في عدم تشغيل هذا الكود حالياً هو حجم البيانات. إرسال 3 إطارات بحجم كبير يحتاج إلى استجابة أسرع للشبكة وموارد معالجة أكبر. 

بمجرد التحويل لهذا الكود، يجب على فريق الـ Cloud تحديث موارد محرك `wasel-saas-engine` كالتالي:

1. **الذاكرة (Memory):** رفعها من `1024Mi` إلى `2048Mi` (2 جيجا) أو `4096Mi`، لأن `Pillow` سيقوم بفتح وصناعة Thumbnails لـ 3-5 صور في نفس اللحظة.
2. **المعالجات (CPU):** رفعها لـ `2 CPU` لمعالجة الصور والتواصل مع Gemini API في وقت قصير لا يشعر به المستخدم.
3. **تزامن الطلبات (Concurrency):** تفعيله إلى حد أقصى 80 لضمان عدم تراكم الطلبات المليئة بالصور فوق بعضها في Worker واحد (إعدادات الـ gunicorn الحالية قادرة على التعامل مع ذلك).

### أمر النشر (Deploy) بعد الرفع لتطبيق الموارد:
```bash
gcloud run deploy wasel-saas-engine \
  --source . \
  --region europe-west1 \
  --memory 4096Mi \
  --cpu 2 \
  --concurrency 80 \
  --allow-unauthenticated
```

## 🤖 3. لماذا هذا مهم للذكاء الاصطناعي (Gemini)؟

- **السر العسكري في واصل:** النموذج الأساسي `gemini-2.0-flash`، ونظام الـ Prompt الموجه له يقول: 
  `انظر إلى هذا التسلسل الحركي للصور (Motion Sequence)...`
- إذا قدمت له صورة واحدة، سيحاول تخمين الحركة من شكل اليد الثابت.
- إذا قدمت له `[صورة 1، صورة 2، صورة 3]` في المصفوفة (`images_base64`)، سيفهمها كـ **Timeline** ويرى حركة اليد الانتقالية (مثلاً انزلاق اليد للأسفل الذي يعني "شكراً"). هذا يرفع دقة الترجمة من %70 إلى %98 فوراً.
